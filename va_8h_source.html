00112 /* 
00113 Overview 
00114 
00115 The VA API is intended to provide an interface between a video decode/encode/display
00116 application (client) and a hardware accelerator (server), to off-load 
00117 video decode/encode/display operations from the host to the hardware accelerator at various 
00118 entry-points.
00119 
00120 The basic operation steps are:
00121 
00122 - Negotiate a mutually acceptable configuration with the server to lock
00123   down profile, entrypoints, and other attributes that will not change on 
00124   a frame-by-frame basis.
00125 - Create a decode context which represents a "virtualized" hardware decode 
00126   device
00127 - Get and fill decode buffers with picture level, slice level and macroblock 
00128   level data (depending on entrypoints)
00129 - Pass the decode buffers to the server to decode the current frame
00130 
00131 Initialization & Configuration Management 
00132 
00133 - Find out supported profiles
00134 - Find out entrypoints for a given profile
00135 - Find out configuration attributes for a given profile/entrypoint pair
00136 - Create a configuration for use by the decoder
00137 
00138 */
